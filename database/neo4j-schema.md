# Neo4j Graph Schema

Knowledge graph schema for the AI assistant. Use this as a reference when writing Cypher queries.

---

## Node Labels

### `Fact`
Facts and information collected by the assistant.

| Property | Type | Description |
|----------|------|-------------|
| `id` | String | UUID (generated by `randomUUID()`) |
| `content` | String | Fact content |
| `importance` | String | `low`, `normal`, `high`, `critical` |
| `createdAt` | String | ISO 8601 timestamp |
| `source` | String | Where the fact came from (optional) |

### `Lesson`
Conclusions and lessons learned by the assistant.

| Property | Type | Description |
|----------|------|-------------|
| `id` | String | UUID |
| `content` | String | Lesson content |
| `category` | String | e.g. `tooling`, `communication`, `workflow` |
| `createdAt` | String | ISO 8601 timestamp |

### `Person`
People mentioned or associated with facts.

| Property | Type | Description |
|----------|------|-------------|
| `name` | String | Name or nickname |
| `phone` | String | Phone number (optional) |
| `relationship` | String | e.g. `owner`, `friend`, `colleague` |
| `createdAt` | String | ISO 8601 timestamp |

### `Event`
Events and meetings.

| Property | Type | Description |
|----------|------|-------------|
| `id` | String | UUID |
| `title` | String | Event title |
| `description` | String | Longer description (optional) |
| `date` | String | ISO 8601 event date |
| `createdAt` | String | When added to the graph |

### `Topic`
Topics and tags for categorization.

| Property | Type | Description |
|----------|------|-------------|
| `name` | String | Topic name (unique) |
| `description` | String | Topic description (optional) |

---

## Relationships

| Relationship | From → To | Description |
|-------------|-----------|-------------|
| `ABOUT` | Fact → Person | Fact concerns this person |
| `TAGGED` | Fact → Topic | Fact is tagged with this topic |
| `INVOLVES` | Event → Person | Event involves this person |
| `KNOWS` | Person → Person | People know each other |
| `RELATED_TO` | Fact → Fact | Facts are related to each other |
| `TRIGGERED` | Event → Lesson | Event led to this lesson |

## Example Cypher Queries

### Add a fact

```cypher
CREATE (f:Fact {
    id: randomUUID(),
    content: "User prefers dark mode",
    importance: "normal",
    createdAt: datetime().epochMillis
})
```

### Add a fact linked to a person

```cypher
MERGE (p:Person {name: "Alice"})
CREATE (f:Fact {
    id: randomUUID(),
    content: "Alice is a senior developer",
    importance: "high",
    createdAt: datetime().epochMillis
})
CREATE (f)-[:ABOUT]->(p)
```

### Get recent facts (context)

```cypher
MATCH (f:Fact)
OPTIONAL MATCH (f)-[:ABOUT]->(p:Person)
OPTIONAL MATCH (f)-[:TAGGED]->(t:Topic)
RETURN f.content, f.importance, p.name AS about,
       collect(DISTINCT t.name) AS topics
ORDER BY f.createdAt DESC
LIMIT 15
```

### Search facts by content

```cypher
MATCH (f:Fact)
WHERE toLower(f.content) CONTAINS toLower("dark mode")
RETURN f.content, f.importance, f.createdAt
ORDER BY f.createdAt DESC
LIMIT 10
```

### Get everything about a specific person

```cypher
MATCH (p:Person {name: "Alice"})
OPTIONAL MATCH (f:Fact)-[:ABOUT]->(p)
OPTIONAL MATCH (e:Event)-[:INVOLVES]->(p)
RETURN p, collect(DISTINCT f) AS facts, collect(DISTINCT e) AS events
```

### Lessons from the last 30 days

```cypher
MATCH (l:Lesson)
WHERE l.createdAt >= datetime() - duration('P30D')
RETURN l.content, l.category, l.createdAt
ORDER BY l.createdAt DESC
```

### Graph statistics

```cypher
MATCH (n)
RETURN labels(n)[0] AS label, count(n) AS count
ORDER BY count DESC
```

### Related facts (find connections through topics)

```cypher
MATCH (f1:Fact)-[:TAGGED]->(t:Topic)<-[:TAGGED]-(f2:Fact)
WHERE f1 <> f2
RETURN f1.content, t.name, f2.content
LIMIT 20
```

## Indexes (recommended)

Run these queries in Neo4j Browser after creating the instance:

```cypher
-- Unique index on Person.name
CREATE CONSTRAINT person_name_unique IF NOT EXISTS
    FOR (p:Person) REQUIRE p.name IS UNIQUE;

-- Unique index on Topic.name
CREATE CONSTRAINT topic_name_unique IF NOT EXISTS
    FOR (t:Topic) REQUIRE t.name IS UNIQUE;

-- Index on date for facts and lessons
CREATE INDEX fact_created_at IF NOT EXISTS FOR (f:Fact) ON (f.createdAt);
CREATE INDEX lesson_created_at IF NOT EXISTS FOR (l:Lesson) ON (l.createdAt);
CREATE INDEX event_date IF NOT EXISTS FOR (e:Event) ON (e.date);

-- Index on fact importance
CREATE INDEX fact_importance IF NOT EXISTS FOR (f:Fact) ON (f.importance);
```

## Tips

- **Merge vs Create**: Use `MERGE` for nodes that should be unique (Person, Topic), `CREATE` for facts and events
- **UUID**: Always generate `randomUUID()` for fact/lesson/event IDs
- **Timestamps**: Store as ISO string (`datetime().epochMillis` or Python `datetime.now().isoformat()`)
- **Backup**: Neo4j Aura automatically creates backups, but you can also export via APOC
